# 拖拽功能使用指南

## ✅ 已完成的改进

### 1. 拖拽功能
- ✅ 项目卡片可以拖动
- ✅ 文件夹可以作为拖放目标
- ✅ 拖动时有视觉反馈（半透明效果）
- ✅ 拖动到文件夹上时有高亮效果（蓝色边框）
- ✅ 拖放完成后自动更新项目位置

### 2. 语义改进
- ✅ "所有项目" 改为 "桌面"
- ✅ 桌面显示所有未分类的项目
- ✅ 文件夹内的项目不会出现在桌面

## 🎯 使用方法

### 拖拽项目到文件夹
1. **点击并按住项目卡片**
2. **拖动到左侧的文件夹上**
3. **文件夹会高亮显示蓝色边框**
4. **松开鼠标完成移动**

### 其他移动方式
1. **使用移动按钮**：点击项目卡片上的绿色箭头图标
2. **编辑项目**：在编辑项目时选择所属文件夹

## 🎨 视觉反馈

- **拖动中**：项目卡片变为半透明（opacity: 0.5）
- **拖动到文件夹上**：文件夹显示蓝色边框和浅蓝背景
- **拖动结束**：项目卡片恢复正常，显示成功提示

## 📝 技术实现

### 拖拽事件
- `@dragstart`: 开始拖动，记录被拖动的项目
- `@dragend`: 结束拖动，清理状态
- `@dragover`: 拖动经过文件夹，显示高亮
- `@dragleave`: 离开文件夹，取消高亮
- `@drop`: 放置到文件夹，执行移动操作

### 状态管理
- `draggedProject`: 当前被拖动的项目
- `dragOverFolder`: 当前鼠标悬停的文件夹

## 🚀 重启应用

如果遇到缓存问题，请：

1. **停止开发服务器**（Ctrl+C）
2. **清除缓存**
   ```bash
   rm -rf node_modules/.vite
   ```
3. **重新启动**
   ```bash
   npm run electron:dev
   ```

## 🐛 故障排除

### 拖拽不工作
1. 确保项目卡片有 `draggable="true"` 属性
2. 检查浏览器控制台是否有错误
3. 重启应用

### 文件夹不高亮
1. 确保 `dragOverFolder` 状态正确更新
2. 检查 CSS 类是否正确应用
3. 查看开发者工具中的元素样式

## 📊 项目组织建议

- **桌面**：放置临时项目或未分类项目
- **文件夹**：按主题、客户或项目类型分类
- **命名规范**：使用清晰的文件夹名称，如"客户项目"、"个人笔记"等
